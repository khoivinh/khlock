## What Was Done This Session (March 24)

### Bug Fixes

- **Sidebar close icon alignment** — The close icon in the sidebar was sitting 7px lower than the drawer-open icon in the header. Root cause: the sidebar header flex container used `items-center`, which vertically centered the close icon against the taller "Login or Sign Up" button. Fixed by switching to `items-start` to match the header's alignment.

- **Mobile timezone/temperature alignment** — The timezone and temperature display on mobile was misaligned from the city name. Root cause: the mobile timezone `<p>` was outside the main flex column with extra `px-2.5` padding. Fixed by moving it inside the `flex-1 min-w-0` column (same parent as city name and desktop time) and removing the padding.

- **Long-press drag too sensitive** — Long-pressing a clock tile sometimes triggered scrolling and sometimes drag-and-drop, making the behavior unpredictable. Fixed by increasing the tile body long-press delay from 350ms to 600ms and tightening the movement tolerance from 8px to 5px. The drag handle icon remains at its faster 150ms setting.

- **Body scrolling behind sidebar** — The main page content scrolled behind the open sidebar overlay. Fixed by adding a `useEffect` that sets `document.body.style.overflow = "hidden"` when the sidebar is open, restoring the previous value on close.

- **Duplicate-key crash on city change** — Changing a clock tile's city to one already displayed elsewhere in the list caused the adjacent tile to turn yellow and the app to become unresponsive. Root cause: `handleZoneChange` had no duplicate key guard (unlike `handleAddClock`), so selecting an existing city created duplicate keys in the React list and dnd-kit's sortable context, corrupting state. Fixed by adding a duplicate check that swaps tile positions instead of creating duplicates.
