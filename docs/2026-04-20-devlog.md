### 2. UX enhancements

- **Scroll-to-current-city on edit** (`digital-clock.tsx`): Added a `useEffect` that runs when the city popover opens, finds the selected `CommandItem` by its `data-testid`, and calls `scrollIntoView({ block: "center" })`. Also guarded the filter list so the currently-selected city is always present even when truncated to 100 results, otherwise scroll-to would silently fail for cities outside the default slice (there are 30k+ cities in the dataset).
- **Native `confirm()` replaced with shadcn Dialog** (`digital-clock.tsx`): The browser's native `confirm("Remove X?")` was rendering as `khlock.pages.dev says: Remove newYork_US?` — now renders as a shadcn `<Dialog>` titled "Happyhour" with `Cancel` / `Remove` actions. ESC + outside-click close via Radix defaults.
- **Badge typography** (`digital-clock.tsx`, desktop + mobile): Next / Prev / Offset badges moved from `text-[7px] font-bold leading-[15px]` to `text-[8px] font-semibold leading-[16px]`. Added `tracking-[0.2px]` to the Next/Prev spans only (per spec the offset badge has no tracking).
- **Clock tile removal animation** (`digital-clock.tsx`): Introduced an `isRemoving` state; when the user confirms removal the tile gets `animate-out fade-out-0 zoom-out-95 duration-200`, then a 200ms `setTimeout` calls the parent's `onRemove` to actually remove the tile from the list. Drops the previous abrupt disappearance.

### 3. Bug fixes

- **Z-index on multi-word city (`Los Angeles`, etc.)**: Bumped the city-selector `PopoverContent` className from implicit `z-50` to `z-[60]` so it always sits above the active-edit tile's yellow-highlight state.
- **Default logged-out second tile showing "Select City"**: Code already specified `["tokyo_JP", "newYorkCity_US", "paris_FR"]`. The bug was stale `newYork_US` in localStorage — the fresh `world-happyhour-*` key prefix resolves it automatically. No code change needed.
- **Idle data-clearing**: Added `visibilitychange` + `focus` listeners on `TimeZoneConverter` that force-refresh `currentTime` when the tab returns to the foreground. Standalone `useEffect` so it doesn't tear down with drag activity or custom-mode toggles.
- **Closest city targets Los Angeles for Northern CA**: Added `client/src/lib/closest-city.ts` with a haversine-distance search over all ~30k cities. On mount, after setting the timezone-derived fallback, calls `navigator.geolocation.getCurrentPosition` and upgrades `heroZone` to the geographically-nearest city. Cached for 24h in `world-happyhour-local-city`. Silent — denial/timeout stays on the old timezone heuristic, so no regression for users who deny permission.
- **Drawer toggle alignment (open vs closed)**: Removed `mt-[2px]` from the sidebar's internal close button so it aligns with the page-header toggle. Both `DrawerToggleIcon` SVGs already share the same viewBox + rect coords; the misalignment came from this one offset class. Tier 2 will swap in the redrawn (thicker-stroke) icons from Figma but the geometry stays consistent.
