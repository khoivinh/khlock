## Investigation findings

### Error State #8 — Invalid time input
- Attempted to reach: submitting an empty `editTime`, out-of-range hours/minutes, non-numeric input.
- Result: unreachable via the native `<input type="time">`. The browser prevents invalid values from ever being set in state, and `handleUpdateClick` (`digital-clock.tsx:248`) has an explicit `if (editTime)` guard on empty strings.
- Decision: dropped from PRD rather than implementing a defensive border + disabled OK.

## Follow-up batch (2026-04-22 1404)

- **Duplicate-city edit: tile A stuck yellow** (`client/src/components/digital-clock.tsx`). Root cause: in `CitySelector`'s `onSelect`, calling `setOpen(false)` only updated Radix's controlled `open` prop — it did NOT trigger the external `onOpenChange` callback (Radix suppresses that for controlled-prop updates from the consumer side). Result: parent's `isDropdownOpen` stayed `true`, keeping the yellow focus background on the editing tile whenever no zone swap happened (i.e. the duplicate-city path). Fix: call `onOpenChange(false)` explicitly alongside `setOpen(false)` in `onSelect`.
- **Happy-mode cycle icon** (`client/src/components/icons/happy-mode-icon.tsx`). Replaced the stroked-smile glyph with the full Happyhour brand mark (yellow circle + black check-eye + black smile) at 24×24, per revised Figma 198:1780. Same SVG paths as `HappyhourLogo` default variant.
