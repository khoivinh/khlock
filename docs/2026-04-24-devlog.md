## 2. Cookie consent UX overhaul

User decided (via AskUserQuestion) on **two re-entry points** for GDPR robustness: a yellow button on `/privacy` (matching Figma 250:4243) AND a "Cookie Preferences" footer link between Privacy and Support. The floating `#stcm-icon` that Silktide spawns after dismissal is now hidden site-wide — Silktide has no config flag for this, so suppressed via a single CSS rule in `client/src/index.css`.

Footer link order is now `About • Privacy • Cookie Preferences • Support`. Implemented as a `<button>` (not a `Link`) since it triggers an action rather than navigating; styled to match existing nav items for visual parity.

Both entry points call a shared `openCookiePreferences()` helper in `client/src/lib/cookie-consent.ts`, which null-chains into `window.silktideConsentManager.getInstance().toggleModal(true)`. A new ambient declaration in `client/src/types/silktide.d.ts` keeps `tsc --noEmit` clean without committing to a full Silktide type definition.

Privacy page copy tweaked: `"opt out anytime via the cookie banner"` → `"manage your preferences anytime below"` (the old sentence referenced a banner that no longer persists). Google Analytics wording kept since the existing sentence already names it.

**Files:**
- `client/src/index.css` — `#stcm-icon { display: none !important }`
- `client/src/lib/cookie-consent.ts` (NEW) — shared modal-open helper
- `client/src/types/silktide.d.ts` (NEW) — ambient window typings
- `client/src/pages/privacy.tsx` — copy tweak + yellow button
- `client/src/components/site-footer.tsx` — insert Cookie Preferences button between Privacy and Support

### GDPR posture

The dual-entry design satisfies Article 7(3) ("as easy to withdraw as to give") — the footer link is one click from any page, and the Privacy page button is the canonical, more prominent control. Consent mode 2.0 is still denied-by-default in `client/index.html`, so nothing fires before the user affirmatively opts in.

## 3. Preferences modal restyle (296:3854)

Implementation is CSS-only. Silktide JS is untouched, which keeps consent storage and gtag flipping intact. New stylesheet `client/src/styles/silktide-overrides.css` is imported from `main.tsx` after `index.css` so it cascades over `public/silktide-consent-manager.css`.

Strategy: override the vendored wrapper's CSS custom properties (`--primaryColor`, `--backgroundColor`, `--textColor`, `--iconColor`, `--fontFamily`) at `#stcm-wrapper`, then layer specific rules for the modal container, typography, toggle geometry, and button shape/padding. Buttons get a full repaint (yellow `#FFD900`, dark text, 10px bold uppercase, 6px radius, 8/10/12 padding). Toggles scale from Silktide's 74×34 default down to the Figma-specified 33×18 pill; OFF/ON text labels hidden since the new spec is a clean pill. Close X recolored to yellow.

Responsive: desktop modal clamps to `min(600px, calc(100vw - 32px))`. Under 480px, padding tightens to 24/20 and the Silktide-attribution footer stacks.

**Font substitution:** Figma specifies Libre Franklin for section labels and body copy. Per user direction (and saved to `memory/feedback_fonts_inter_substitution.md` for future sessions), every Libre Franklin reference was substituted with Inter at the same size/weight/leading. No new Google Fonts imports.
