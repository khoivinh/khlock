## Session 2 — Figma-precision iteration (post-`52e7d64`)

User flagged that the cookie preferences modal close-X and toggles weren't faithful to Figma 296:3854 — "the X is too big, the toggles are a different style altogether." Set the bar going forward: when a Figma design is shared, recreate it precisely; no substitutions or alterations without clearing first. Saved that as `feedback_figma_precision.md` so the next session inherits it.

### Round 1 — modal precision (`872fc1a`)

**Close X** — Silktide ships a 20×20 thick-X SVG at `padding: 13px`, total ~46×46. Figma's spec is two 3×17 rounded rects (`rounded-2`, fill `#FFD900`) rotated ±45° in a 14.142×14.142 box. Hid Silktide's SVG and rebuilt the X via `::before` / `::after` pseudo-elements at the exact Figma geometry.

**Toggles** — generic filled-pill replaced by *outlined-pill + small circular knob*, exact Figma construction: 1.5px stroke, knob `r=5` (10×10).
- OFF: white stroke + white knob centered at `cx=9` (left)
- ON: `#22C55E` stroke + green knob centered at `cx=24` (right)
Source: extracted from the Figma `Switch Stroke` SVGs (toggle nodes 101:1184 / 114:1132). Hid Silktide's `OFF`/`ON` text labels.

**Silktide attribution row** — Figma 296:3854 has the Silktide-S icon next to the credit text in the modal footer, but Silktide's modal markup omits the SVG (only the banner footer renders it). Injected via `mask-image` on `.stcm-credit-link::after` with a URL-encoded SVG data URI — color follows `background-color` so it matches the credit text.

**Modal action button label** — `text.preferences.saveButtonText: 'Accept all'` (Figma label). Pairs with the existing `'Essential Only'` from `text.prompt.rejectNonEssentialButtonText`. Semantic note: Silktide's "save" applies whatever the toggle state currently is, so when paired with "Accept all" the user must toggle Analytics on first if they want to grant — acceptable since both consent types are visible.

**Spacing** — header gap-9 + margin-bottom 10, fieldset margin 0 0 15 with last fieldset margin-bottom 30, consent-row `align-items: flex-start` (was `center`). Matches Figma's "Top" container `flex-col gap-15 pb-30`.

**Mobile banner description** — reset desktop's `flex: 1 1 0` (was leaking into the column-flex on `<640px`), set `width: 100%`, switched line-height from `1.4` to `normal` to match Figma `leading-[normal]`.

### Round 2 — banner row layout, backdrop dim, spacing bump (`e254029`)

User: "Cookie consent UI still does not look correct. It's center-aligned where the Figma layout left-aligns the text and keeps the two buttons side-by-side." Plus: please dim the page like the sidebar does when the modal opens, and the spacing above "Necessary" still feels insufficient (per Figma 301:3986, the narrow-width variant the user added).

**Mobile banner left-align** — root cause: Silktide's default `.stcm-actions { flex-direction: column }` below 600px. Buttons stacked full-width and got their built-in `text-align: center` — that's the "centered stacked" look. Fix: force `flex-direction: row` + `justify-content: flex-start` at all widths, with `flex-wrap: nowrap` on mobile. Buttons now stay side-by-side and left-aligned per Figma 280:5123.

**Modal backdrop dim** — Silktide already supports a backdrop element (`#stcm-backdrop`), but our config never enabled it (`createBackdrop` is gated on `config.backdrop.show === true`). Enabled it in `client/index.html`, overrode `--backdropBackgroundColor` to `rgba(0, 0, 0, 0.3)` to match the sidebar's `bg-black/30` exactly. Side effect: the bottom banner would also have gotten the dim. Suppressed via `#stcm-wrapper:has(#stcm-banner) #stcm-backdrop { display: none !important }` — Silktide's `removeBanner()` strips the node entirely when the modal opens, so `:has` stops matching and the inline `display: block` from `showBackdrop` takes over. Net behavior: bottom banner appears without backdrop; preferences modal appears with the same dim as the sidebar.

**Modal spacing above Necessary** — bumped description `margin-bottom` 15 → 25px so the description's descenders don't crowd the next legend at narrow widths (per Figma 301:3986 rendering). Also reset Silktide's default `flex: 1` on `#stcm-modal section` to `flex: 0 0 auto` so it doesn't try to grow in the modal's column-flex.
